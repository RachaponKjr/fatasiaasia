1. Executive Summary

Platform Overview and Key Features

1.1 Platform Overview

Fantasia Asia is a comprehensive tour booking platform designed to connect travelers with curated Asian tour experiences. The platform enables users to browse tours, create bookings, and manage their travel plans, while providing administrators with powerful tools to manage tours, process bookings, and handle payments.

Key Components

Component Technology Purpose
User Website Next.js 15, React 18, TypeScript Customer-facing booking platform
Admin Dashboard React 18, Vite, JavaScript Backoffice management system
User API Go, Gin Framework User authentication & tours
Admin API Go, Gin Framework Admin operations & payments
Database PostgreSQL 15+ Data persistence
Storage DigitalOcean Spaces Images and files

1.2 Key Features

👤 User Features
  • Browse and search tours by country, category, and duration
  • User registration with email verification (OTP)
  • Secure JWT authentication
  • Wishlist management
  • Tour booking with traveler details
  • Profile management with avatar upload
  • Booking history and cancellation
⚙️ Admin Features
  • Complete tour CRUD operations
  • Booking confirmation and pricing
  • Payment link generation via Omise
  • Client management
  • Tour agency account creation
  • File upload to cloud storage

2.2 Data Flow: Booking Process

  1. Browse: User visits website → Next.js fetches tours from User API
  2. Book: User submits booking → User API creates booking in database
  3. Notify: Admin receives email notification about new booking
  4. Confirm: Admin confirms booking → Admin API creates Omise payment link
  5. Pay: User clicks payment link → Completes payment on Omise
  6. Update: Omise sends webhook → Admin API updates booking to "paid"

2.3 Authentication Flow

JWT-Based Authentication
  1. User submits credentials to /auth/login
  2. API validates credentials against database
  3. On success, API generates JWT token (1-hour expiry)
  4. Token returned to client, stored in cookies
  5. Client includes token in Authorization header for protected requests
  6. API validates token on each request

3.2 Authentication

Registration Flow

  1. User enters email, password, first name, last name
  2. System sends 6-digit OTP to email
  3. User enters OTP (5-minute expiry)
  4. Account created on successful verification
Rate Limits:
  • Max 3 OTP requests per email per 15 minutes
  • Max 5 verification attempts per OTP

Login Flow

  1. User enters email and password
  2. System validates credentials
  3. JWT token issued (1-hour expiry)
  4. Token stored in cookies

GET /tour

List all visible tours. Optional country query parameter.

GET /tour/:id

Get detailed tour information including itinerary and gallery.
